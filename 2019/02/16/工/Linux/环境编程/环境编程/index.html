僵尸进程

进程完成工作后，没有被销毁即被称作“僵尸进程”，进程的终止有两个方式

  • 传递参数并调用 exit 函数
  • main函数中执行 return 语句并返回值

但是返回的值传递给操作系统后，操作系统并不会销毁子进程，直到将这些值传递给该子进程的父进程

wait

可以利用 wait 函数获取子进程的返回值，但是如果没有已终止的子进程，那么程序将阻塞(Blocking)直到有子进程终止。

waitpid

还可以调用 waitpid 函数，而且并不会引起阻塞

signal

但是主进程不可能只调用 waitpid 函数以等待子进程终止，所以需要可以是用 signal 函数来传递有子进程终止的信号，实际上很少使用 signal ，只是为了保证对旧程序的兼容，而是使用sigaction

sigaction

还可以是用 sigaction 替代 signal，因为 signal 可能在不同的 UNIX 系统中有区别，但 sigaction 完全相同
